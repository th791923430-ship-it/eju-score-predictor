# 逐校官网补齐流程（文科优先）

## 目标
- 让 学校名称/学部/学科 的展示命名与各大学官方外国人留学生入试页面一致。
- 优先补齐文科相关学部在以下关键字段：EJU必要科目、英语要求、出愿日期、校内考/面试、合格通知。
- 保持预测模型数据源不变：录取概率仍使用 cleaned_eju_data.csv（そうがくしゃ历史数据）。

## 一次补齐的标准动作（每校）
1. 打开该校 募集要项一览链接，确认是否存在“外国人留学生/私費外国人留学生”入口。
2. 进入学部级 PDF（或学部页面），先记录该校允许报考的学部清单。
3. 对文科相关学部逐条核对：
   - 学部/学科正式名称（以官网记法为准）
   - EJU必要科目（是否含 総合科目、数学コース1）
   - 英语要求（是否必须、认可考试类型）
   - 出愿开始/截止日期（含二期）
   - 选考方式（书类、校内考、面试）
4. 回填 CSV：优先填 具体学部PDF、EJU必要科目、是否需要英语成绩、英语成绩类型、出愿开始日期、出愿结束日期、第二次选考/校内考/面试 内容。
5. 若学校名或学部名带注记（例如 括号说明），仅保留官方正式名称到 学校名称/学部/学科；注记放备注或内部说明，不写入名称字段。
6. 填写 更新时间、填写人，并将 是否更新到最新 标为 1。
7. 完成 5-10 所学校后，重跑校验清单并处理新增高优先级。

## 文科优先判断规则
- 文科高优先关键词：文/法/経済/経営/商/社会/国際/人文/教育/心理/政策/観光/教養/コミュニケーション。
- 若 EJU必要科目 含 総合科目 或 数学コース1，直接归为文科优先。
- 若同一学部同时接受文理（both），按文科优先处理，因为规则差异通常更复杂。

## 清单工具用法
1. 运行：python3 generate_official_fill_checklist.py
2. 产出文件：official_fill_checklist_bunka_priority.csv
3. 按 priority_rank 从小到大推进（1 为最高优先级）。
4. 每完成一批学校后再次运行，直到高优先级学校 missing_total 明显下降。

## 质量闸门（提交前）
- 学校层：官方存在的留学生可报考学部已全部在 CSV 中出现。
- 字段层：文科学部关键字段不为空（EJU、英语、出愿、选考、合格通知）。
- 命名层：学校名与学部名不含临时说明性括号注记。
- 系统层：/api/schools 与官方 CSV 抽取对齐（miss/extra 维持 0 或可解释）。

## 建议节奏
- 每日两轮：上午补齐，下午回归校验。
- 单轮建议 15-25 所学校，先处理 priority_score 高的学校。
